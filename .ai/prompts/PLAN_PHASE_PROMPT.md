# Prompt: Plan Phase

Purpose: given a product brief or objective, create a phase plan and split it into implementation slices.

## Instructions

1. Read `.ai/AI_CONTRACT.md`, `.ai/STATE.md`, and `.ai/CONTEXT_INDEX.md`.
2. Read the relevant brief or objective.
3. Ask only blocking questions.
4. Prefer safe assumptions only when they are reversible and documented.
5. Create a small phase with clear non-goals and acceptance criteria.
6. Split the phase into small slices that can be implemented independently.
7. Ensure each slice names required reads, likely files, allowed dependencies, validation commands, and required state updates.
8. Update `.ai/ROADMAP.md`.
9. Update `.ai/CONTEXT_INDEX.md` if new context categories or important files are introduced.

## Output

- Phase file path and summary.
- Slice file paths and summaries.
- Updated roadmap notes.
- Updated context index notes.
- Open questions and assumptions.
