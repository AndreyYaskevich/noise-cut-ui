# Roadmap

This roadmap tracks the staged delivery of NoiseCut Web App v0.

| Phase | Goal | Status | Notes |
| --- | --- | --- | --- |
| Phase 0 | Align AI-delivery-kit artifacts and initialize repo-local planning workflow. | Completed | Docs/state/task setup only. No runtime app code yet. |
| Phase 1 | Define Founder-lens Product Signal Report contracts and additive API integration shape. | Proposed | Keep API changes thin and backward-compatible. |
| Phase 2 | Add Founder-lens report generation through the existing API. | Proposed | Preserve extension compatibility. |
| Phase 3 | Scaffold the React + TypeScript web app shell. | Proposed | No framework has been chosen yet. |
| Phase 4 | Build the New Report and Report Result experience. | Proposed | Founder lens first. |
| Phase 5 | Add extension handoff into the web app experience. | Proposed | Avoid large payloads in query strings. |
| Phase 6 | Add report feedback and privacy-safe telemetry. | Proposed | Prefer existing API telemetry patterns. |
| Phase 7 | Add private beta access controls and beta-readiness docs. | Proposed | Lightweight invite or beta token only. |
| Phase 8 | Capture paid-validation signals without building full billing. | Proposed | Validation-first, not monetization infrastructure. |
